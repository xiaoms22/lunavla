# Visual attribution and evidence boundary

The current v1.1 README does not use real PushT, ACT, or Diffusion Policy media as evidence. Pre-correction result cards and external reference GIFs are isolated under `docs/archive/v1.0/media/`; they were not generated by the runnable NumPy point-reach core and must not appear in current comparisons or release evidence packs.

The only files remaining under `images/` are ecosystem context assets. Their origins are recorded in `images/ecosystem/source_manifest.json`. They may illustrate the broader robotics landscape, but they do not establish a LunaVLA capability and are not referenced by the current README result section.

Publishable v1.1 visuals must be generated from a validated v1.1 manifest, identify the source rollout hash, and remain within the synthetic `pusht_style_point_reach` claim boundary.
